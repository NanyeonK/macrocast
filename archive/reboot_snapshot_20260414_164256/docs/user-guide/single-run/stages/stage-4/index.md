# Stage 4 — Evaluation

Stage 4 measures forecast quality.

## What Stage 4 owns

- point forecast metrics
- relative metrics
- benchmark comparison setup
- aggregation over time, horizons, and targets
- reporting style for forecast performance

## Main question

How do we decide whether a forecasting path performed well?

## Separation rule

Evaluation is about metric computation and aggregation.
Formal statistical tests belong to Stage 6, not here.

## Current status

The package already exposes evaluation components, but the stage-first documentation still needs to be expanded beyond the current transitional pages.
